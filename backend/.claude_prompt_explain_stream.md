# 修复 /api/v1/ai/explain-stream 端点

## 问题
`POST /api/v1/ai/explain-stream` 当前直接返回 `StreamingResponse`（直接 SSE 流），但前端期望两步模式：
1. POST `/ai/explain-stream` → 返回 `{"task_id": "..."}`
2. GET `/api/v1/tasks/{task_id}/events` → 返回 SSE 流

前端通过 `useTaskEvents(taskId)` hook 连接第二步的 SSE 端点。

## 目标
改造 `backend/app/api/v1/routes/ai_routes.py` 中的 `explain_stream` 路由：
1. 接收 JSON body：`{"question_id": "...", "depth": "standard"}`
2. 调用 `service.explain_question_stream()` 创建任务（这已经返回 `task_id`）
3. 返回 `{"task_id": str(task_id)}` 的 JSON 响应
4. **不要返回 `StreamingResponse`** — SSE 事件通过已有的 `/api/v1/tasks/{task_id}/events` 端点消费

- `backend/app/api/v1/routes/task_events.py` — 已有 `/api/v1/tasks/{task_id}/events` 端点

## 约束
1. **只修改 `explain_stream` 路由**，不要改其他端点
2. **不要改 `explain_question_stream` 服务方法** — 它已经正确创建任务并返回 `(task_id, event_gen)`

4. 前端通过 JSON body 传参（`Body()`），不是查询参数
