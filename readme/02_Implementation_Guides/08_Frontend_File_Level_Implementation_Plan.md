# 08_Frontend_File_Level_Implementation_Plan

## 1. 目标

本文件是给 Claude Code 的前端逐文件修改清单，目标是在**现有 Next.js 前端基础上做增量增强**，重点支持：

- 简历驱动面试入口
- 分页与数据加载优化
- 统一加载态 / 空态 / 错误态
- 简历来源可视化
- 简历模式模拟面试
- 题库、学习、统计页面的结构化增强

原则：
- 不推倒重做
- 不破坏现有路由结构
- 优先复用现有组件
- 优先做通用能力，再做专项 UI
- 所有改动必须兼容现有后端和未来扩展

## 3. 文件级实施清单

### 3.1 `web/src/app/layout.tsx`

#### 目标
为全局页面提供统一导航、布局与基础状态承载能力。

#### 调整建议
- 保持现有布局不变
- 增加全局顶部导航中的“简历驱动面试”入口
- 可预留全局通知区或任务状态区
- 如有统一侧边栏，可预留“简历训练”菜单项

#### 注意事项
- 不要把业务逻辑写到 layout 里
- 不要让 layout 直接请求大量数据
- 仅做全局壳层与导航增强

---

### 3.2 `web/src/app/page.tsx`

#### 目标
首页作为统一入口，应成为“简历驱动面试”的首站。

#### 调整建议
- 新增一个主功能卡片：`简历驱动面试`
- 新增一个次级卡片：`最近简历`
- 新增一个次级卡片：`继续上次面试`
- 最近导入区按分页或小批量展示，避免全量拉取
- 首页只展示概览，不承载明细列表

#### 需要新增的信息
- 简历导入数量
- 最近生成题数
- 当前待复习题数
- 最近一次简历训练状态

#### 推荐交互
- 点击 `简历驱动面试` 进入导入页或面试页
- 点击最近简历进入简历详情/解析结果
- 点击继续上次面试恢复 session

---

### 3.3 `web/src/app/import/page.tsx`

#### 目标
导入页扩展为统一导入中心，支持题目导入和简历导入。

#### 调整建议
- 保留现有文本导入与文件上传
- 新增一个 `简历导入` Tab 或区域
- 简历导入支持：
  - 拖拽上传
  - 文件选择
  - 粘贴简历文本
- 简历导入成功后展示结构化结果：
  - 技术栈摘要
  - 项目经历摘要
  - 可出题方向
  - 解析状态
  - 下一步动作

#### 需要新增的 UI 状态
- `简历上传中`
- `简历解析中`
- `简历结构化中`
- `简历出题中`
- `简历导入成功`
- `简历导入失败`

#### 推荐动作按钮
- 查看解析结果
- 基于简历生成题目
- 直接开始模拟面试
- 进入题库查看相关题

#### 注意事项
- 文件上传和简历上传逻辑尽量复用
- 导入后不要只给一个成功提示，要给下一步入口

---

### 3.4 `web/src/app/questions/page.tsx`

#### 目标
题目列表页是未来数据规模最大的页面之一，必须做分页和来源筛选。

#### 调整建议
- 增加分页组件，默认每页 20 或 30 条
- 增加来源筛选：
  - 简历生成
  - 文件导入
  - 文本导入
  - 手动录入
  - AI 生成
- 增加简历相关筛选：
  - 仅看简历题
  - 仅看项目题
  - 仅看追问题
- 保留现有搜索和领域、难度筛选

#### 每条题目卡片建议新增信息
- 来源类型
- 是否来自简历
- 关联技术栈
- 关联知识节点数量
- 是否已练习
- 是否已进入复习计划

#### 分页行为
- 切换筛选条件时回到第一页
- 搜索时重置分页偏移
- 翻页时保留筛选条件
- 空页时给出“没有当前页数据”的提示，而不是整页空白

#### 注意事项
- 不要一次拉 200 条固定展示
- 列表页要和后端分页接口完全对齐

---

### 3.5 `web/src/app/questions/[id]/page.tsx`

#### 目标
题目详情页负责展示来源证据、知识链路和 AI 讲解，是解释能力最强的页面。

#### 调整建议
- 增加来源信息区
- 增加简历证据区
- 增加知识节点区
- 增加“为什么出这道题”区
- 增加 AI 讲解区的折叠/展开
- 增加“开始模拟面试”“继续追问”“重新生成相似题”按钮

#### 推荐展示模块
1. 题目基本信息
2. 来源与证据链路
3. 技术栈映射
4. 关联知识节点
5. AI 讲解
6. 用户练习记录
7. 相似题推荐

#### 交互建议
- 默认只显示摘要
- 点击“展开来源证据”查看简历原文片段
- 点击“开始模拟面试”跳转到 `/interview`

#### 注意事项
- 不要让 AI 讲解和来源证据混在一起
- 明确区分“事实”与“生成内容”

---

### 3.6 `web/src/app/interview/page.tsx`

#### 目标
在保留通用面试能力的同时，增加“简历模式”入口。

#### 调整建议
- 增加面试模式切换：
  - 通用模式
  - 简历模式
  - 混合模式
  - 深挖模式
- 简历模式下先选择简历或项目经历，再开始出题
- 显示当前题目的来源、相关技术栈、相关项目
- 面试结束后输出与简历相关的薄弱点总结

#### 推荐 UI 改造
- Setup 阶段新增“简历选择器”
- 新增“项目选择器”
- 新增“技术栈选择器”
- 新增“模式说明卡片”
- 新增“切换回通用模式”按钮

#### 注意事项
- 保留当前无简历模式的完整可用性
- 简历模式必须可以无缝落到现有 session 流程里
- 不要单独再创建一套新页面

---

### 3.7 `web/src/app/study/page.tsx`

#### 目标
学习页是练习与复习中心，应成为简历驱动训练的重要落点。

#### 调整建议
- 练习模式增加“简历来源题目优先”开关
- 练习模式增加“按简历项目抽题”
- 复习模式支持按来源筛选：
  - 简历题
  - 题库题
  - 错题
- 学习统计区增加来源分布和简历维度

#### 建议增强点
- 题目来源徽标
- 技术栈标签
- 项目追问标签
- 简历相关薄弱项面板
- 练习列表分页化

#### 注意事项
- 练习题列表不要一次全量加载
- 复习列表也要分页或分批加载
- 讲解和评价卡片应明确区分 AI 结果与用户结果

---

### 3.8 `web/src/app/stats/page.tsx`

#### 目标
统计页应该以聚合指标为主，明细记录分页展示，避免全量明细拉取。

#### 调整建议
- 统计概览保留聚合指标
- 最近学习记录改为分页或小批量加载
- 增加来源维度统计：
  - 简历相关
  - 题库相关
  - 错题相关
- 增加技术栈维度统计
- 增加项目维度统计

#### 推荐展示
- 简历导入次数
- 简历派生题目数量
- 简历相关练习轮次
- 高频薄弱技术栈
- 最近复习状态

#### 注意事项
- 不要在统计页拉 100 条以上原始记录直接渲染
- 统计页应更多依赖后端聚合接口

## 5. 建议的修改顺序

### 第一阶段
1. 首页增加简历入口
2. 导入页增加简历导入
3. 题目列表页加分页与来源筛选
4. 题目详情页加来源证据区

### 第二阶段
5. 模拟面试页加简历模式
6. 学习页加按简历筛选与分页
7. 统计页加来源维度和分页

### 第三阶段
8. 抽公共组件
9. 统一状态组件
10. 统一分页与筛选组件

---

## 6. 给 Claude Code 的执行要求

- 优先修改已有文件，不要额外创建大量重复页面
- 每次新增功能都要考虑分页和状态态
- 所有新入口都要兼容简历模式与通用模式
- 所有列表类页面默认分页
- 所有 AI 结果必须在视觉上与原始数据区分
- 尽量抽公共组件，减少页面内重复代码
