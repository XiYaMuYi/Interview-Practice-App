## Enhancement Goals

### Apply consistently across ALL pages:
- Use `.soft-card` for all card containers instead of inline `rounded-xl border bg-white shadow-md`
- Use `.page-title` and `.page-subtitle` for page headers
- Use `.section-title` and `.section-hint` for section headers
- Use `.primary-chip` and `.secondary-chip` for tags/badges
- Add subtle hover transitions to interactive cards
- Use the existing color variables (var(--primary), var(--accent), etc.) instead of hardcoded colors
- Ensure cards have `soft-card-hover` class for lift effect on hover
- Use consistent spacing (gap-4, gap-6, p-4, p-6) instead of arbitrary values

### Page-specific enhancements:

**Home page (`page.tsx`):**
- Make it feel like a "learning cockpit"
- Stats overview cards should use `.soft-card` + hover effect
- Feature grid cards should have subtle icon + clear CTA
- "Continue learning" primary button should stand out
- Recent activity section with clean list styling
- Add visual hierarchy: overview → features → recent activity

**Import page (`import/page.tsx`):**
- Clear tab distinction between "Import Questions" and "Import Resume"
- Upload area should have drag-drop visual feedback (dashed border, hover state)
- Parse status flow should be clean and sequential
- Results section should use cards for previews
- Clear "next step" guidance after each action

**Question list (`QuestionsPageClient.tsx`):**
- Filter bar should use consistent chip styling
- Question cards should be clean and scannable
- Difficulty badges, domain tags, source badges should be unified
- Pagination should sit cleanly at bottom
- Empty/error/loading states should be centered and friendly

**Question detail (`questions/[id]/page.tsx`):**
- Layered presentation: question → source → knowledge → hints → explanation
- Section dividers should be subtle
- "Continue to next question" and "Start mock interview" CTAs should be prominent
- Use `.soft-card` for each section block

**Study page (`study/page.tsx`):**
- Practice mode and review mode should be visually distinct tabs
- Progress indicators should use the accent color
- Question cards should feel actionable (want to click and practice)
- Stats summary at top should use summary cards
- Empty states should encourage action

**Interview page (`interview/page.tsx`):**
- Focused, immersive feel
- Input area should be clean and prominent
- Conversation history should have clear speaker distinction
- Scoring and feedback should stand out but not be overwhelming
- Progress through interview should be visible

**Stats page (`stats/page.tsx`):**
- Dashboard feel: metrics first
- Charts should be clean and readable
- Emphasize trends and improvement, not just numbers
- Use accent color for positive trends, warning for areas needing attention
- Cards should group related metrics

## Critical Rules
1. **INCREMENTAL ONLY** — Do NOT rewrite pages from scratch. Read each file, then make targeted edits.
2. **NO INLINE STYLES** — Use existing CSS classes from globals.css wherever possible. Add new classes to globals.css only when truly needed.
3. **DO NOT CHANGE FUNCTIONALITY** — Only visual/CSS changes. Don't change data fetching, API calls, or business logic.
4. **DO NOT TOUCH** `layout.tsx` or `globals.css` unless you need to ADD new utility classes (not modify existing ones).
5. **Tailwind only** — Do not introduce new CSS frameworks or libraries.
6. **After all changes**: Run `npm run build` and ensure it passes with zero errors. Fix any issues iteratively.

## Badge System
Use these existing badge components consistently:
- `SourceBadge` for question source (简历/AI/文本/文件)
- Difficulty indicators: use chip colors — easy (green/success), medium (yellow/warning), hard (red/danger)
- Domain tags: use primary-chip style
- Question type: use secondary-chip style
