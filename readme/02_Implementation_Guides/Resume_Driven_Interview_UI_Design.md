# Resume_Driven_Interview_UI_Design

## 1. 目标

本页面组的目标不是单纯增加“简历上传”按钮，而是把简历驱动出题能力无缝融入现有 `Interview-Practice-App` 的主流程中，形成以下闭环：

1. 上传简历
2. 解析简历并提取技术栈、项目经历、职责与成果
3. 自动生成与简历强相关的面试题
4. 进入模拟面试或题目浏览
5. 支持讲解、追问、评分、复盘、错题沉淀

该能力必须兼容现有题库、导入、学习练习、模拟面试、统计等页面，不能成为一条孤立链路。

---

## 2. 现有前端页面现状判断

基于当前代码，现有前端已经具备以下页面：

- `首页`：入口聚合、展示最近题目、功能导航
- `导入页`：支持文本导入与文件上传
- `题目列表页`：搜索、筛选、浏览
- `模拟面试页`：按领域与难度开始面试
- `学习页`：刷题与练习
- `统计页`：学习数据查看

### 2.1 现状结论

**结论：前端具备承接简历功能的基础，但目前入口还没有细化到“简历驱动出题”这一专门路径。**

也就是说：
- 后端架构层面已覆盖简历能力
- 前端现有导入页可直接承接“简历上传”
- 但需要补一层交互设计，让用户明确知道“这是简历驱动面试入口”

---

## 3. 前端交互设计原则

### 3.1 三个核心入口

前端必须新增或微调为三个核心入口：

1. **简历上传入口**
   - 允许用户上传 PDF / DOCX / TXT / MD 简历
   - 作为首页和导入页的主要入口之一

2. **简历驱动出题入口**
   - 用户上传简历后，可一键进入“基于简历生成题目”流程
   - 支持选择出题风格：技术栈题 / 项目追问题 / 综合面试题

3. **简历模拟面试入口**
   - 从简历题目列表页或详情页直接进入模拟面试
   - 支持从“当前简历题目”连续追问，不再只依赖全局领域选择

### 3.2 交互目标

- 用户能一眼看懂：这是“简历面试训练”，不是普通题库导入
- 上传后马上看到解析结果与出题预览
- 用户可以决定：先看题目、直接练习、还是让系统继续深挖
- 交互链路尽量短，避免跳转过多

## 4. 建议新增页面 / 区块

### 4.1 首页增加“简历训练”主卡片

在首页功能卡片中新增：

- **简历训练**
  - 描述：上传简历，自动生成专属面试题
  - 跳转：`/resume` 或直接复用 `/import` 中的简历 tab

首页应同时保留：
- 导入题目
- 浏览题目
- 学习练习
- 模拟面试
- 学习统计

这样可保证新能力嵌入主流程，不破坏现有导航。

### 4.2 导入页增加“简历导入”Tab

当前 `ImportPage` 已有文本导入与文件上传，建议在同页加入三段式 Tab：

- **题目导入**
- **简历导入**
- **知识资料导入**

其中“简历导入”区域包含：
- 文件上传
- 简历用途说明
- 解析进度提示
- 生成题目入口按钮

### 4.3 新增“简历面试工作台”页面

建议新增独立页面：`/resume-interview` 或 `/resume/[id]`

页面结构建议：
- 左侧：简历摘要与技术栈标签
- 中间：生成的面试题列表 / 当前面试题
- 右侧：出题策略控制、难度控制、追问深度控制

这个页面是简历驱动能力的核心工作台。

## 5. 页面级交互设计清单

## 5.1 首页 `/`

### 目标
把“简历驱动面试”作为主入口之一显性展示。

### 交互项
- 新增“简历训练”功能卡
- 卡片点击后进入简历导入页
- 最近导入区增加“简历”类型筛选提示
- 首页 Quick Stats 可增加“简历数量 / 简历生成题数 / 简历训练次数”

### 微调建议
- 首页首屏 hero 文案增加“上传简历，自动生成面试题”
- 功能卡片顺序建议：导入题目、简历训练、学习练习、模拟面试、学习统计

---

## 5.2 导入页 `/import`

### 目标
把“简历上传”放到现有导入体系里，用户无需切换新页面就能开始。

### 交互项
- 增加 Tab：`题目导入 / 简历导入 / 资料导入`
- 简历导入区支持拖拽与点击上传
- 上传后显示：
  - 简历解析状态
  - 提取到的技术栈数量
  - 提取到的项目经历数量
  - 可直接生成的题目数
- 上传完成后提供两个主按钮：
  - `查看生成题目`
  - `立即开始模拟面试`

### 微调建议
- 当前文件上传结果区域要支持区分“题目文件”和“简历文件”
- 上传卡片要让用户知道它是“简历解析入口”而不是普通文件导入
- 文件类型提示可以保留，但增加简历专用说明：
  - “支持 PDF / DOCX 简历，系统将提取技术栈、项目经历和职责描述”

---

## 5.3 题目列表页 `/questions`

### 目标
让用户能看到“由简历生成的题目”和“普通题库题目”的区别。

### 交互项
- 新增筛选维度：
  - 来源：简历生成 / 手动导入 / 知识资料生成
  - 关联简历：某份简历
  - 关联项目：某个项目经历
- 列表项增加来源标签：
  - `简历题`
  - `项目追问`
  - `技术栈题`
- 支持从题目列表直接进入模拟面试

### 微调建议
- 每个题目卡片显示“来源证据”入口
- 支持快速查看“这道题来自简历哪一段”

---

## 5.4 题目详情页 `/questions/[id]`

### 目标
把简历证据、题目、讲解、追问、练习串起来。

### 交互项
- 显示题目来源类型
- 显示关联简历片段 / 关联项目经历
- 增加按钮：
  - `讲解这道题`
  - `模拟面试追问`
  - `加入错题集`
  - `查看关联简历`
- 展示“推荐追问点”

### 微调建议
- 详情页要有“简历原文证据”折叠区
- 对于项目题，展示项目名、职责、技术栈、成果四块信息
- 对于技术栈题，展示关联知识节点和前置知识

---

## 5.5 模拟面试页 `/interview`

### 目标
将“领域模拟面试”扩展为“简历驱动模拟面试”和“通用模拟面试”两种模式。

### 交互项
新增模式切换：
- `通用模式`
- `简历模式`

#### 通用模式
当前已有：
- 选择领域
- 选择难度
- 开始面试

#### 简历模式
新增：
- 选择简历
- 选择简历中的项目
- 选择出题范围：技术栈 / 项目 / 综合
- 选择追问深度：浅问 / 标准 / 深挖

### 微调建议
- 简历模式下，开始前应展示简历摘要和将要考察的主题
- 开始后，首题应来自简历内容，而不是随机领域题
- 结束后展示“基于简历的能力画像”

---

## 5.6 学习页 `/study`

### 目标
把错题复习、简历题练习和知识点复习统一到一个学习闭环。

### 交互项
- 显示今日待复习题目
- 区分题目来源：简历题 / 普通题 / 错题
- 支持“按简历项目复习”
- 支持“按技术栈复习”

### 微调建议
- 增加“来自简历的薄弱点”模块
- 增加“项目追问专项训练”入口

---

## 5.7 统计页 `/stats`

### 目标
让用户看到简历驱动训练的效果。

### 交互项
新增统计维度：
- 简历上传次数
- 简历生成题数
- 项目追问次数
- 技术栈覆盖率
- 简历面试平均分
- 薄弱技术栈排行

### 微调建议
- 增加“简历能力画像”图表
- 可用标签云展示高频技术栈
- 可用条形图展示项目题命中率

## 6. 建议的前端微调策略

### 6.1 不推翻现有页面，只做增强

现有页面已经有较好的骨架，建议策略是：

- **首页**：加入口
- **导入页**：加简历 Tab
- **题目页**：加来源与简历过滤
- **面试页**：加简历模式
- **统计页**：加简历训练指标

这样最符合“现有基础上进行设计”的原则。

### 6.2 前端状态建议

建议补充以下全局状态：
- 当前简历 ID
- 当前简历摘要
- 当前项目经历
- 当前出题模式（通用 / 简历）
- 当前题目来源
- 当前追问深度

### 6.3 组件建议

建议新增组件：
- `ResumeUploadCard`
- `ResumeSummaryPanel`
- `ResumeExperienceList`
- `ResumeQuestionSourceBadge`
- `ResumeInterviewModeSwitch`
- `ResumeQuestionPreview`
- `ResumeCoverageStats`

## 7. 与后端联动的最小接口清单

前端要完整支持该功能，建议至少依赖以下接口：

- `POST /api/resumes/upload`
- `POST /api/resumes/extract`
- `GET /api/resumes`
- `GET /api/resumes/{id}`
- `GET /api/resumes/{id}/experiences`
- `POST /api/resumes/{id}/generate-questions`
- `GET /api/questions?source_type=resume`
- `POST /api/ai/interview/start`（支持 resume_id）
- `POST /api/ai/interview/answer`

## 8. 页面流程建议

### 流程 A：简历上传后直接出题

1. 首页 / 导入页点击“简历训练”
2. 上传简历
3. 展示解析结果
4. 点击“生成面试题”
5. 进入题目预览
6. 点击“开始模拟面试”

### 流程 B：先看题，再面试

1. 上传简历
2. 系统生成题目列表
3. 用户筛选项目题 / 技术栈题
4. 点击某题进入详情
5. 点击“模拟面试”

### 流程 C：直接进入简历模拟面试

1. 选择简历
2. 选择项目
3. 选择深度
4. 点击“开始面试”
5. 系统按简历证据提问并追问

---

## 9. 最终结论

现有前端**已经具备承接简历功能的基础**，但还需要做一层“简历驱动面试”的交互增强，才能让 Claude Code 更容易按当前页面体系继续开发，而不是重做一套新的前端逻辑。

推荐落地方式是：
- **不重构现有页面**
- **在现有页面上增加简历模式与简历入口**
- **新增少量专用组件和一个简历工作台页**
- **让简历上传、出题、模拟面试形成完整闭环**
