# Token 级真流式改造

## 背景
项目已经完成了 Phase 1-6 的流式任务基础设施（SSE、TaskManager、事件推送等），但当前多数 `*_stream` 方法内部使用的是阻塞式 `chat()` / `chat_json_with_prompt()`，属于"伪流式"——先把完整结果生成完，再一次性发 SSE 事件。

现在要把最关键的 3 个接口改成 **token 级真流式**，让前端能实时看到 AI 逐字生成的效果。

## 项目路径
D:\AI_Project\Surprise\Interview-Practice-App

## 现状分析

### LLMGateway 已有能力
`backend/app/infra/llm/gateway.py` 中已有：
- `stream_chat(messages, ...)` → `AsyncGenerator[str, None]` — 已调用 `provider.stream_chat_completion()`，**真流式**

### 需要改造的方法（按优先级）

#### 优先级 1: `explain_question_stream` (ai_service.py)
**现状**: 调用 `self.llm.chat([...])` 阻塞等待完整结果，然后 `yield _sse("content", {"content": response})`
**目标**: 改用 `self.llm.stream_chat([...])`，每个 token 都 yield `token` 事件，最后 yield `result` 事件

#### 优先级 2: `handle_interview_turn_stream` (ai_service.py)
**现状**: 
- `self._evaluate_single_answer()` 阻塞
- `self.llm.chat_with_prompt("interview_summary", ...)` 阻塞
- `self.generate_followup()` → `chat_json_with_prompt()` 阻塞
**目标**: 
- `interview_summary` 改用 `stream_chat_with_prompt`，逐 token 输出
- `generate_followup` 可以保留 JSON 模式（结构化输出不适合 token 流）
- 评价阶段至少做 progress 事件流

#### 优先级 3: `generate_review_summary_stream` (ai_service.py)
**现状**: `self.llm.chat_with_prompt("review_summary", ...)` 阻塞
**目标**: 改用 `stream_chat_with_prompt`，逐 token 输出

## 改造方案

### 1. LLMGateway 新增方法

在 `backend/app/infra/llm/gateway.py` 中新增：

```python
async def stream_chat_with_prompt(
    self,
    prompt_key: str,
    *,
    variables: dict[str, str] | None = None,
    temperature: float = 0.7,
    max_tokens: int = 2048,
    task_id=None,
    session_id: str | None = None,
) -> AsyncGenerator[str, None]:
    """从 Prompt 模板构建消息并以流式方式输出。
    每个 chunk 直接 yield，支持 token 级流式。
    """
    template = prompt_registry.get_template(prompt_key)
    if template is None:
        raise LLMProviderError(self.provider_name, f"Prompt '{prompt_key}' not found in registry")
    
    system_content = template.system_template
    if variables:
        for key, value in variables.items():
            system_content = system_content.replace(f"{{{{{key}}}}}", value)
    
    messages = [{"role": "system", "content": system_content}]
    # 直接委托给已有的 stream_chat
    async for chunk in self.stream_chat(
        messages,
        temperature=temperature,
        max_tokens=max_tokens,
        prompt_key=prompt_key,
        prompt_version=template.version,
        task_id=task_id,
        session_id=session_id,
    ):
        yield chunk
```

### 2. 改造 explain_question_stream

将阻塞调用替换为流式调用，在事件 generator 中：
- progress 事件照常发
- 用 `stream_chat()` 替代 `chat()`
- 每收到一个 token，yield `_sse("token", {"token": chunk})`
- 同时累积完整文本
- 最后 yield `_sse("content", {"content": full_text})` 和 `_sse("result", {"content": full_text, "depth": depth})`
- 保存到 DB（question.explanation = full_text）
- yield `_sse("done", ...)`

### 3. 改造 handle_interview_turn_stream

- `evaluate` 阶段：保留 JSON 模式（评价需要结构化输出），但增加 progress 中间事件
- `summary` 阶段：改用 `stream_chat_with_prompt("interview_summary", ...)`，逐 token yield
- `followup` 阶段：保留 JSON 模式（结构化输出）

### 4. 改造 generate_review_summary_stream

- 改用 `stream_chat_with_prompt("review_summary", ...)`
- 逐 token yield `token` 事件
- 最后 yield `content` + `done` 事件

### 5. SSE 事件格式

统一新增 `token` 事件类型：
```
event: token
data: {"task_id": "...", "token": "这是逐字输出的文本片段"}
```

原有事件保持不变：
- `progress` — 阶段状态
- `content` — 完整内容（一次性）
- `result` — 结构化结果
- `evaluation` — 评价结果
- `followup` / `summary` — 追问/总结
- `done` — 完成
- `error` — 错误

## 约束
1. **保持向后兼容** — 同步版本（`explain_question`、`handle_interview_turn`、`generate_review_summary`）不改动
2. **TaskManager 保持工作** — 任务进度、状态更新照常
3. **缓存策略** — token 流过程不缓存，最终结果可以缓存
4. **日志** — `stream_chat` 已有日志收集（collect 完整响应后异步写日志），不要重复
5. **异常处理** — try/except 块保持，异常时更新 task status 为 failed
6. **不要改前端代码** — 本次只改后端，前端后续独立处理
