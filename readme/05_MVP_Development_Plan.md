# 05_MVP_Development_Plan

## 1. MVP 目标拆解

### MVP 1：最小可用版
目标是先跑通“导入资料 → 自动整理 → 可浏览可检索”的闭环。

核心能力：
- 文本粘贴
- 文件上传
- 题目识别
- 自动分类
- 标签管理
- 列表展示
- 搜索筛选
- 题目详情页

### MVP 2：智能学习版
目标是在 MVP 1 基础上增加“讲解、排序、复习、对话”的能力。

核心能力：
- 难度排序
- 学习路径推荐
- 答案分层生成
- 错题集
- 学习记录
- 模拟问答
- LangGraph 智能体基础流程

### 1.1 架构演进基调

本项目采用 **模块化单体** 路线。MVP 的任务优先级不是“最快把功能堆出来”，而是“先把边界定好，再逐步扩展”。

#### 必须遵守的实现顺序
1. 先搭 `LLMGateway` 与 `PromptRegistry`
2. 再搭 `services/` 业务编排层
3. 再写 `infra/` 的数据库、文件、解析、向量适配
4. 再写 `api/` 路由
5. 最后接入前端页面

#### 强制约束
- API 不能直接调大模型
- API 不能直接写数据库查询
- 前端不能依赖未稳定的内部模型结构
- 所有 AI 调用必须先经过统一网关
- 核心表必须预留扩展字段，不能因 MVP 简化掉

---

## 2. 总体开发优先级

建议按照以下顺序开发：

1. 后端数据模型与基础 API
2. 文件上传与文本导入
3. 题目抽取与分类
4. 前端题目列表与详情页
5. 搜索与筛选
6. 标签与错题功能
7. AI 讲解与难度排序
8. 智能对话与模拟面试
9. 学习记录与复盘

## 8. 交付标准

- 能导入文本或文件
- 能自动生成题目并分类
- 能在前端查看题库
- 能搜索、筛选、查看详情

### MVP 2 完成标准
- 能一键讲解题目
- 能模拟面试追问

- 能输出错题与复盘建议
- 能形成基本学习闭环

---

## 9. 风险与应对

### 风险 1：文件解析质量不稳定
应对：优先支持文本粘贴，文件解析作为增强能力。

### 风险 2：AI 输出不稳定
应对：统一结构化输出，所有结果写入 schema。

### 风险 3：功能过多导致延期
应对：先做 MVP 1，再逐步加智能学习与对话。

### 风险 4：页面复杂度上升
应对：前端采用轻量统一组件库，先保证信息清晰。
