# 12_UI_Redesign_Instructions_For_Agent

## UI 全局视觉升级执行指令

本文件用于指导自动编码 Agent 在现有前端基础上统一升级 UI 风格。目标不是炫技，而是建立一种“清晰、克制、专业、愿意学习”的视觉气质，让页面能激发刷题欲望和学习欲望。

---

## 1. 设计目标

### 1.1 风格基调

- 统一风格
- 漂亮但不华丽
- 轻盈、安静、专业
- 具有学习工作台气质
- 能触发学习欲望和答题欲望

### 1.2 关键原则

- 不使用过度夸张的霓虹、重金属、赛博风
- 不使用花哨的强动画破坏专注
- 不用过深的纯黑背景制造压迫感
- 不让卡片和按钮各自为政
- 所有页面视觉应该像同一个产品体系，而不是拼接页面

---

## 2. 全局视觉系统约束

### 2.1 颜色体系

推荐采用：
- 主色：清爽蓝 / 青蓝
- 辅色：青绿 / 靛蓝
- 中性色：Slate 灰系
- 背景：偏冷的浅灰白、柔和渐变背景

### 2.2 组件语义

- 卡片统一圆角和边框
- 按钮统一主次层级
- 标签统一胶囊风格
- 状态统一 badge 风格
- 页面容器统一最大宽度和留白节奏

### 2.3 气质目标

页面要传达：
- 我可以开始学
- 我知道下一步做什么
- 我正在变强
- 我能看到自己的进步

---

## 3. 全局布局要求

### 3.1 `layout.tsx`

- 顶部导航改为轻量半透明浮层风格
- 品牌区要更有识别感
- 导航项必须统一视觉状态
- 移动端需保留简化入口
- 主内容区要使用柔和背景承托

### 3.2 页面容器

- 所有页面使用统一的最大宽度与内边距
- 页面内容外层应有柔和卡片承托
- 页面标题、说明、模块区块要统一层级

---

## 4. 全局样式要求

### 4.1 `globals.css`

必须统一定义：
- 背景基调
- 字体系统
- 卡片样式
- 悬浮态
- 圆角规范
- 阴影规范
- 胶囊标签规范
- 页面主框架样式

### 4.2 背景风格

推荐采用：
- 柔和渐变背景
- 微弱光斑/气泡感
- 白色半透明卡片
- 轻微玻璃拟态，但不能影响可读性

### 4.3 交互动效

- hover 要柔和、克制
- 过渡速度统一
- 不使用夸张弹跳动画
- 动效服务于提示，不破坏内容阅读

---

## 5. 学习欲望与答题欲望设计

### 5.1 反馈感

- 每完成一步都给出明确反馈
- 进度、分数、掌握度要可视化
- 最近练习、待复习、下一题要显著呈现

### 5.2 目标感

- 用户必须始终知道“下一步做什么”
- 每个页面都应提供一个清晰主行动按钮

### 5.3 成长感

- 页面应让用户看到自己的提升
- 学习记录、错题、掌握度、复习时间应能形成节奏感

---

## 6. 实施优先级

1. 先统一 `layout.tsx` 和 `globals.css`
2. 再统一卡片、按钮、Badge、空态、加载态
3. 再逐个页面做轻量视觉增强
4. 最后再补更复杂的统计图与画像页

---

## 7. 给自动编码 Agent 的执行指令

当你开始做 UI 重构时，必须先保证全局视觉基调一致，再做页面细节。不要在不同页面各自创造一套风格。

如果你发现某个页面的样式与全局风格不一致，必须先收敛到统一设计系统，而不是继续往该页面堆样式。
