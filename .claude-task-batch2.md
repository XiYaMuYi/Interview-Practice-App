## 背景
当前 `web/src/app/study/page.tsx` 只有空壳"功能开发中"。后端 API 已全部实现：

**后端 API 清单：**
- `GET /api/v1/questions?offset=0&limit=20&domain_type=&difficulty_level=` — 获取题目列表
- `GET /api/v1/questions/:id` — 获取题目详情
- `POST /api/v1/ai/evaluate` — 评分 { question_id, user_answer } → { score, feedback, missing_points, is_pass, mastery_level }
- `POST /api/v1/study/records` — 创建学习记录 { question_id, study_type, user_answer, ai_score, ai_feedback, mastery_level }
- `POST /api/v1/study/review` — 记录复习 (SM-2) { question_id, quality(0-5), user_answer }
- `GET /api/v1/study/review-list` — 待复习列表
- `GET /api/v1/study/stats` — 学习统计
- `POST /api/v1/ai/explain` — AI 讲解 { question_id, depth }

## 任务

### 实现 `web/src/app/study/page.tsx`

一个完整的单页面学习应用，包含以下功能：

**1. 页面布局（三栏或标签页）：**
- 标签 1：「练习模式」— 随机/选择题目 → 用户作答 → AI 评分 → 反馈
- 标签 2：「待复习」— 展示需要复习的题目列表（SM-2 调度）
- 标签 3：「学习统计」— 展示总练习数、平均分、掌握题目数等

**2. 练习模式：**
- 从 `/api/v1/questions` 加载题目列表（可指定 domain_type / difficulty_level 过滤）
- 展示一道题目，用户在文本框中输入回答
- 点击"提交回答"：
  a. 调用 `POST /api/v1/ai/evaluate` 获取 AI 评分
  b. 调用 `POST /api/v1/study/records` 保存学习记录
  c. 展示评分结果：分数 (0-100)、反馈、遗漏要点
  d. 显示"查看讲解"按钮，可调用 explain API
- "下一题"按钮进入下一道

**3. 待复习列表：**
- 从 `GET /api/v1/study/review-list` 获取待复习题目
- 每道题目显示：标题、难度、下次复习时间
- 点击可进入答题界面（同练习模式）
- 完成后调用 `POST /api/v1/study/review` 记录质量评分（0-5 按钮）

**4. 学习统计：**
- 从 `GET /api/v1/study/stats` 获取数据
- 展示卡片：总练习次数、总复习次数、平均分数、已掌握题目数、待复习题目数
- 使用简洁的卡片布局

### 技术要求

- 完善的 loading 和错误状态
